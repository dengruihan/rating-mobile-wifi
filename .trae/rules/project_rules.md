# 移动WiFi评级网站前端开发规则

## 项目概述
本项目旨在开发一个WiFi评级网站的完整前端原型，让用户能够浏览、搜索和评价不同地点的WiFi服务质量。

## 开发目标
- 创建一个可交互的移动WiFi评级网站前端原型，实现所有用户面向的功能
- 使用模拟JSON数据代替真实后端API
- 实现完整的用户流程和页面导航

## 核心要求

### 1. 交互性要求
- 所有UI元素必须可点击且有响应
- 每个按钮、链接、操作都应引导至预期页面或行为
- 模拟所有数据交互，不连接真实服务器

### 2. 模拟数据要求
- 使用JSON格式的模拟数据
- 所有显示的信息、用户操作和动态内容都应由模拟数据驱动
- 模拟数据应反映真实的数据库结构

### 3. 页面开发要求
必须开发以下页面：
- **登录/注册页面**
  - 用户登录表单
  - 用户注册表单
  - 密码重置功能

- **首页**
  - 网站介绍和主要功能入口
  - WiFi热点推荐列表
  - 搜索功能

- **用户仪表板/个人资料**
  - 用户个人信息管理
  - 用户评价历史
  - 收藏的WiFi热点

- **移动WiFi型号页面**
  - 可筛选和排序的移动WiFi型号列表
  - 每个型号显示基本信息和评分

- **移动WiFi型号详情页面**
  - 型号详细信息（信号强度、速度、设备价格、资费等）
  - 用户评价列表
  - 提交评价的表单

- **搜索结果页面**
  - 基于用户搜索条件的结果列表
  - 过滤和排序选项

### 4. 数据库设计
- 确保所有用户交互都有数据库支持
- 模拟JSON数据应反映真实的数据库结构
- 数据库模式应匹配实际数据流和页面要求

## 技术栈
请优先使用VUE进行前端开发，确保所有页面都能正常交互。
- HTML5
- CSS3 (可使用CSS框架如Bootstrap)
- JavaScript (可使用前端框架如React、Vue等)
- JSON (用于模拟数据)
